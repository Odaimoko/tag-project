# Tag Project - 新手快速上手指南

## 🎉 欢迎使用 Tag Project！

这份指南将帮助您在几分钟内开始使用 Tag Project 管理您的任务。

## 📋 前置要求

1. 已安装 Obsidian
2. 已安装 Dataview 插件（必需）
3. 已安装并启用 Tag Project 插件

## 🚀 三种快速开始方式

### 方式一：快速开始向导（推荐新手）

这是最简单的方式，系统会引导您完成初始设置。

1. **打开命令面板**
   - 按 `Ctrl+P` (Windows/Linux) 或 `Cmd+P` (Mac)

2. **启动向导**
   - 输入 "Quick Start Wizard"
   - 选择该命令

3. **按照向导操作**
   - 了解什么是 workflow
   - 选择预设的 workflow 模板
   - 点击完成

**预设模板包括：**
- 📝 **开发工作流**：todo → doing → review → done
- 🔍 **研究工作流**：identify → investigate → summarize → apply
- ✅ **审查工作流**：content, grammar, formatting, accuracy（可任意顺序）

### 方式二：可视化配置管理器

如果您想自定义 workflow，使用配置管理器。

1. **打开 Manage Page**
   - 点击左侧边栏的 📖 图标
   - 或使用命令面板：`Ctrl+P` → "Open Manage Page"

2. **点击"Manage Workflows"按钮**
   - 在页面顶部可以看到蓝色的按钮

3. **创建您的 Workflow**
   - **名称**：给 workflow 起个名字（如：feature_dev）
   - **类型**：
     - Chain：步骤按顺序完成
     - Checkbox：步骤可任意顺序完成
   - **步骤**：用逗号分隔，如：`planning, coding, testing`
   - **项目**（可选）：所属项目名称

4. **点击"Create"完成创建**

### 方式三：手动创建（高级用户）

如果您熟悉 markdown，可以直接编辑配置文件。

1. **创建或打开配置文件**
   - 在 vault 根目录创建 `tag-project-config.md`

2. **添加 workflow 定义**
   ```markdown
   # 我的工作流配置
   
   - [ ] my_workflow #tpm/workflow_type/chain #tpm/step/todo #tpm/step/doing #tpm/step/done
   ```

3. **刷新数据库**
   - 回到 Manage Page
   - 点击"Refresh"按钮

## 📝 创建第一个任务

创建完 workflow 后，就可以在任何 markdown 文件中创建任务了！

### 示例 1：使用开发工作流

```markdown
## 今天的任务

- [ ] 实现用户登录功能 #tpm/workflow/development
- [ ] 修复注册页面的 bug #tpm/workflow/development
- [ ] 优化数据库查询性能 #tpm/workflow/development
```

### 示例 2：使用研究工作流

```markdown
## 学习计划

- [ ] 学习 React Hooks #tpm/workflow/research
- [ ] 研究微服务架构 #tpm/workflow/research
```

### 示例 3：为任务分配项目

```markdown
## 项目任务

- [ ] 设计登录界面 #tpm/workflow/development #prj/auth_system
- [ ] 实现密码加密 #tpm/workflow/development #prj/auth_system
```

## 🎯 在 Manage Page 中管理任务

创建任务后，打开 Manage Page 就能看到所有任务的统一视图。

### 功能特性

1. **按 Workflow 分组**
   - 所有任务按 workflow 分类显示
   - 一目了然地看到每个任务的进度

2. **筛选功能**
   - 按项目筛选
   - 按标签筛选
   - 按状态筛选
   - 按模块筛选

3. **快捷操作**
   - 点击任务跳转到原文件
   - 右键菜单快速操作
   - 批量修改任务状态

4. **可视化步骤**
   - 看到每个任务当前在哪个步骤
   - 一键切换到下一步骤

## 💡 使用技巧

### 1. 设置快捷键

建议为常用命令设置快捷键：

- 打开设置 → 快捷键 → 搜索 "Tag Project"
- 推荐设置：
  - `Ctrl+Shift+M`：Open Manage Page
  - `Ctrl+Shift+W`：Set Workflow
  - `Ctrl+Shift+P`：Set Project

### 2. 使用命令面板快速操作

在编辑任务时，光标放在任务行上，使用命令：

- **Set Workflow**：为当前任务分配 workflow
- **Set Project**：为当前任务分配项目
- **Set Priority**：设置任务优先级
- **Jump To Manage Page**：跳转到该任务在 Manage Page 中的位置

### 3. 右键菜单

在编辑器中右键点击任务，可以快速：
- 设置 workflow
- 设置项目
- 设置优先级
- 跳转到 Manage Page

### 4. 批量操作

选中多行任务文本后，右键菜单会出现批量操作选项。

## 🔧 常见问题解决

### 问题 1：看不到任何 workflow

**解决方案：**
1. 确认 Dataview 插件已启用
2. 点击"Refresh Manage Page"按钮
3. 检查配置文件格式是否正确

### 问题 2：任务没有显示在 Manage Page

**可能原因：**
- workflow 标签拼写错误（检查是否为 `#tpm/workflow/your_workflow_name`）
- 任务不是 markdown 任务格式（必须是 `- [ ]` 开头）
- workflow 还没有定义

### 问题 3：无法创建 workflow

**检查项：**
- Workflow 名称是否包含特殊字符（只允许字母、数字、下划线、连字符）
- 是否至少定义了一个步骤
- 是否与现有 workflow 重名

### 问题 4：配置文件在哪里？

配置文件 `tag-project-config.md` 位于 vault 的根目录。如果找不到，可以：
1. 打开 Manage Workflows
2. 点击"Open Config File"按钮
3. 系统会自动打开或创建该文件

## 📚 进阶使用

### 自定义 Workflow 类型

目前支持两种类型：

1. **Chain（链式）**
   - 适合有明确顺序的工作流
   - 任务会按步骤顺序推进
   - 示例：todo → doing → review → done

2. **Checkbox（复选框式）**
   - 适合独立的检查项
   - 各步骤可以任意顺序完成
   - 示例：代码审查的多个检查点

### 组织大型项目

对于大型项目，建议：

1. **使用项目标签**
   ```markdown
   - [ ] 任务 #tpm/workflow/development #prj/main_project
   - [ ] 子任务 #tpm/workflow/development #prj/main_project/sub_module
   ```

2. **利用模块功能**
   - 使用 markdown 标题组织任务
   - 在设置中启用"Header as Module"

3. **善用标签过滤**
   - 为任务添加自定义标签
   - 在 Manage Page 中按标签筛选

### 优先级管理

1. 在设置中配置优先级标签
2. 使用"Set Priority"命令为任务设置优先级
3. 在 Manage Page 中按优先级排序
